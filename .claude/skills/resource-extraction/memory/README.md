# Run-memory for the resource-extraction skill

Cross-run learnings — the stuff that isn't in the reference because it was
discovered the hard way.

**At the start of a run:** read `LEARNINGS.md`.

**At the end of a run:** append to `LEARNINGS.md` ONLY if you hit something new
and reusable — a novel artifact data-shape, a transfer channel that behaved
differently, a schema/tooling quirk, a content trap. Rules to keep it lean:

- One or two dated bullets, max. Terse.
- Reusable only. NOT: "did subject X", routine successes, per-resource specifics
  (counts, titles, PR numbers) — those live in the PR/git history, not here.
- If it's a permanent fact about the pipeline, promote it into
  `reference/extraction-and-conversion.md` instead of logging it here.
- Prefer editing/merging an existing bullet over adding a near-duplicate.

The founder explicitly does not want this clogged. When in doubt, don't log.

This folder can be committed (learnings ride along in the run's content PR) or
gitignored for friction-free updates — founder's choice.
